# API 사용 정책

## 서론
NUNCHI API는 관계 분석 및 코칭 서비스에 대한 프로그래밍 방식의 액세스를 제공합니다. API 사용은 본 문서에 명시된 이용 약관의 적용을 받습니다.

## 인증
모든 API 요청은 HTTP Authorization 헤더에 Bearer 토큰을 전송하여 인증해야 합니다. API 키는 등록 시 발급되며 안전하게 보관해야 합니다.

## 사용 제한
시스템 안정성과 공정한 리소스 할당을 보장하기 위해 계정 등급에 따라 속도 제한이 적용됩니다:
- Standard: 분당 60회 요청
- Professional: 분당 300회 요청
- Enterprise: 계약에 따른 맞춤형 제한

이러한 제한을 초과하면 429 (Too Many Requests) 응답 코드가 반환됩니다.

## 데이터 처리
개발자는 NUNCHI 개인 정보 보호 정책을 준수해야 합니다. API를 통해 추출된 원시 오디오 데이터를 외부 서버에 저장하는 것은 대화에 관련된 모든 당사자의 명시적인 동의를 얻지 않는 한 엄격히 금지됩니다.

## 오류 처리
API는 표준 HTTP 상태 코드를 사용합니다:
- 200/201: 성공
- 400: 잘못된 요청
- 401: 승인되지 않음
- 403: 금지됨 - 권한 거부
- 404: 리소스를 찾을 수 없음
- 500: 내부 서버 오류

## 버전 관리
API 버전은 URL 경로에 지정됩니다 (예: /api/v1). 호환되지 않는 변경 사항은 이전 버전에 대해 최소 6개월의 사용 중단 기간을 두고 새로운 버전 식별자 뒤에 도입됩니다.
